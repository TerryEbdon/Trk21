# Security Policy

## Supported Versions

Only the [latest release] is actively supported with security updates.

## Reporting a Vulnerability

1. Go to this repository's [security page][secPageDef]
2. Click the `Report a vulnerability` button. This requires you to be logged
   into a GitHub account.
3. Include:
    - Steps to reproduce
    - Impact assessment
    - Potential fixes or mitigations (if known)
  
**Please do not report security vulnerabilities through public GitHub issues.**

We will do our best to respond within 5 business days and work with you to resolve
the issue.

## Responsible Disclosure

Please allow time for the vulnerability to be investigated and addressed before
disclosing it publicly.

## Security Updates

Security fixes will be released as soon as possible after validation and
testing, and will be announced in the release notes.
